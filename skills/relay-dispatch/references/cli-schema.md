# CLI Flag Schema

Relay CLI scripts declare every supported flag in `scripts/cli-schema.js`. Each value flag has one of two read modes:

- `parsed`: the next token is a value only when it does not look like another flag. A following `--...` token, or a registered reserved token such as `-h`, means the value is missing and the caller fallback is returned.
- `verbatim`: the next argv token is the value exactly as supplied. `--...`, reserved-looking tokens, unicode, quotes, and shell metacharacters stay data, but blank and whitespace-only values are rejected. The fallback is returned only when the flag is absent or is the final argv token.

Presence flags still declare `mode: "parsed"` so the registry can audit all flags uniformly, but they never consume a value.

## Reader Contract

Use `getArg(args, flag, fallback, { commandName })`, `hasFlag(args, flag, { commandName })`, or `getPositionals(args, commandName)` from `scripts/cli-args.js`. That module is only a compatibility wrapper; all reads delegate to `cli-schema.js`.

The reader throws `CliSchemaError` when a caller asks for an unregistered flag or for a flag that is not registered on the selected command. Do not add local `args.indexOf(...)` reads for new flags.

Both `--flag value` and `--flag=value` forms are supported. For parsed flags, a `--`-prefixed value in either form is treated as missing.

## When To Pick Verbatim

Pick `verbatim` when the value is operator-supplied text or a path that should be recorded or resolved exactly:

- prompts and prompt files
- branch names
- audit reasons and skip reasons
- test commands recorded in execution evidence
- local file paths or worktree paths supplied by the operator
- titles, topics, and other user-facing text

Pick `parsed` when the value is a structured selector, enum, number, or generated relay identifier:

- booleans and presence flags
- timeouts and numeric thresholds
- PR numbers and round counts
- executor/reviewer/model selectors
- state, verdict, and merge-method selectors
- run/request/leaf IDs that are generated by relay

Defaults belong at the call site, not in the global schema, because the same flag can have different defaults in different tools.
